\begin{abstract}
\noindent Este documento presenta SurtGIS, una librer\'ia geoespacial de alto rendimiento en Rust con 136 algoritmos y 88 subcomandos CLI, desarrollada como preparaci\'on para el postdoctorado. SurtGIS reduce el tiempo de procesamiento geoespacial por cuenca de 2--3 d\'ias a $\sim$1 hora, permitiendo enfocar el postdoctorado en modelamiento y an\'alisis en vez de ingenier\'ia de datos. El paper describiendo la librer\'ia est\'a en revisi\'on en \textit{Environmental Modelling \& Software} (Elsevier, IF 4.9).
\end{abstract}

\subsection{88 subcomandos CLI}

Ejemplos representativos:

\begin{lstlisting}[language=bash]
# Todos los factores de terreno desde un DEM
surtgis terrain all dem.tif --outdir ./factors/

# Pipeline hidrologico completo
surtgis hydrology all dem.tif --outdir ./hydro/

# Composite cloud-free desde Planetary Computer
surtgis stac composite \
  --collection sentinel-2-l2a --asset red \
  --datetime 2024-01-01/2024-12-31 \
  --bbox -71.5,-28.0,-69.0,-26.0 \
  --max-scenes 12 composite_red.tif

# Indice espectral con expresion arbitraria
surtgis imagery calc \
  --expression "(NIR - Red) / (NIR + Red)" \
  --band NIR=nir.tif --band Red=red.tif ndvi.tif
\end{lstlisting}

\subsection{STAC Composite: diferenciador \'unico}

Un solo comando ejecuta el pipeline completo de composici\'on satelital:
\begin{enumerate}[nosep]
\item B\'usqueda en Planetary Computer/Earth Search (paginaci\'on autom\'atica)
\item Agrupaci\'on por fecha de adquisici\'on
\item Mosaic espacial por fecha (une tiles de la misma pasada)
\item Cloud-mask con SCL (resampleo autom\'atico 20m$\to$10m)
\item Mediana temporal para composite cloud-free
\end{enumerate}

Validado en producci\'on: 150M celdas, 14 tiles, 5 fechas, 10 minutos. \textbf{Ninguna otra herramienta CLI ofrece esto sin Python.}

\section{Impacto en el postdoctorado}

\subsection{Sin SurtGIS (enfoque tradicional)}

Para cada cuenca: instalar GDAL + GRASS + WhiteboxTools, escribir scripts Python para descargar datos, calcular factores de terreno con 3 herramientas distintas, integrar outputs con formatos/CRS incompatibles, cloud-mask manual, composite con numpy/scipy.

\textbf{Tiempo estimado}: 2--3 d\'ias por cuenca $\times$ 15 cuencas = 30--45 d\'ias solo en procesamiento.

\subsection{Con SurtGIS}

\begin{lstlisting}[language=bash]
# DEM: descargar y generar todos los factores
surtgis stac fetch-mosaic --collection cop-dem-glo-30 \
  --bbox ... dem.tif
surtgis terrain all dem.tif --outdir factors/ --compress
surtgis hydrology all dem.tif --outdir hydro/ --compress

# Sentinel-2: composite por banda
for band in red green blue nir swir16; do
  surtgis stac composite --collection sentinel-2-l2a \
    --asset $band --datetime 2024-01-01/2024-12-31 \
    --bbox ... ${band}_composite.tif
done

# Landscape metrics
surtgis landscape analyze landcover.tif --output-csv metrics.csv
\end{lstlisting}

\textbf{Tiempo por cuenca}: $\sim$1 hora (dominado por red, no por c\'omputo). \textbf{15 cuencas}: 2--3 d\'ias, automatizable con un loop bash.

\subsection{Reducci\'on de tiempo: 10--15$\times$}

El postdoctorado se enfoca en \textbf{modelamiento de susceptibilidad}, no en procesamiento de datos. SurtGIS elimina la etapa de procesamiento como bottleneck.

\section{Contribuci\'on cient\'ifica dual}

El postdoctorado produce dos tipos de output:

\begin{enumerate}
\item \textbf{Papers de dominio} (susceptibilidad de humedales, erosi\'on, riesgo) --- donde SurtGIS genera los datos de entrada.
\item \textbf{Paper de software} (EMS, en revisi\'on) --- donde SurtGIS es la contribuci\'on principal.
\end{enumerate}

Los papers de dominio citan a SurtGIS, generando tracci\'on para el paper de software. El paper de software valida la herramienta con los casos de uso del postdoctorado. Ciclo virtuoso.

\section{Conclusi\'on}

SurtGIS transforma el procesamiento geoespacial de un bottleneck de semanas en una operaci\'on de horas. Para el postdoctorado, esto significa dedicar el tiempo al modelamiento y an\'alisis ---~donde est\'a la contribuci\'on cient\'ifica~--- en vez de a la ingenier\'ia de datos que cada investigador resuelve ad-hoc con scripts fr\'agiles.

La herramienta ya existe, est\'a validada, y tiene un paper en revisi\'on. El postdoctorado la usa, la extiende, y la cita.
